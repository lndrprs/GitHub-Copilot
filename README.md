C1.1 - Introdução
 > - GitHub Copilot é um assistente de AI para codificação;
 >   - O assistente ajuda a escrever código e com menos esforço - permitindo o humano focar em resolução de problemas e colaboração; 
 >   - Há uma pesquisa informando que o Copilot melhora a produtividade do desenvolvedor, e acelera o desenvolvimento de software. 
 > - Copilot pode ser usado para:
 >   - Sugestões de códigos enquanto digita na IDE; 
 >   - Ajudar no seu código;
 >   - Pedir ajuda usando a linha de comando;
 >   - Organizar e compartilhar contexto com o Copilot Spaces, para ter mais respostas relevantes;
 >   - Gerar descrições e mudanças em um Pull Request;
 >   - Trabalhar em mudanças de códigos e criar Pull Request para você revisar (Apenas Pro+, Business e Enterprise). 
 >
 > - Copilot pode ser usado em:
 >   - IDE;
 >   - GitHub Mobile;
 >   - Windows Terminal Canary;
 >   - Linha de Comando;
 >   - Site do GitHub. 

C1.2 - Planos 
 > - Free: Acesso limitado;
 > - Pro: Mais flexível - Conclusões limitadas, modelos premiuns, Coding agent, concessão mensal de requisições premiums;
 > - Pro+: Maior level de acesso para desenvolvedores individuais, acesso total a modelos no Chat e maior concessão de requisições;
 > - Business: Agente de código, e permite gerenciamento centralizado, e controle de políticas para os membros da organização;
 > - Enterprise: Possui todas as características do Business, e permite gerenciar organizações, ou usuários e times diretamente. 
 >
 > - Maiores informações: https://docs.github.com/en/copilot/get-started/plans

C1.3 - Características do Copilot
 > - Sugestões Inline: Sugestões que auto-completam o código em IDE's suportados; 
 > - Chat: Permite perguntar questões relacionadas a código; 
 > - Coding Agent: Agente que pode fazer alterações de código para você;
 > - Coding Agents Terceiros: Utilização de agentes terceiros, junto com o Agente do GitHub;
 > - CLI: Interface de linha de comando que permite usar o Copilot dentro do terminal;
 > - Code Review: Revisões de código sugeridas para ajudar a escrever melhor o código;
 > - Pull Request Summaries: Resumos de mudanças que foram feitas em pull request, arquivos impactados e no que o revisor pode focar;
 > - Edits: Realiza mudanças em múltiplos arquivos diretamente de um prompt. Há dois modos:
 >   - Edit: Mais controle granular sobre edições que o Copilot propõe;
 >   - Agente: Para tarefas específicas, e permite o Copilot editar o código de forma autônoma. 
 > - Custom Instructions: Aprimora as repostas do chat, providenciando detalhes de contexto nas suas preferências, ferramentas e requisitos;
 > - Memory: Pode deduzir e armazenar informações úteis sobre repositório, onde o agente e revisador pode usar para melhorar o output nesse repositório;
 > - Desktop: Automaticamente gera mensagens de commit e descrições com o GitHub Desktop;
 > - Spaces: Organiza e centraliza conteúdo relevante - como código, documentos, especificações e mais;
 > - Spark: Constrói e provisiona aplicações Full-Stack usando prompts de linguagem natural.

C1.4 - Características do Copilot para Administradores
 > - Gerenciamento de Política: Gerencia políticas para o Copilot na sua organização ou empresa; 
 > - Gerenciamento de Acesso: Especifica quais organizações na empresa podem usar o Copilot, e quais membros das organizações também;
 > - Dados de uso: Permite gerenciar o acesso e direcionar a adoção do Copilot, com os dados de uso da organização / empresa; 
 > - Audit Logs:  Logs de auditoria na organização para entender quais ações foram feitas e por quais usuários;
 > - Exclude Files: Configura o Copilot para ignorar certos arquivos - útil para não disponibilizar no Copilot.
